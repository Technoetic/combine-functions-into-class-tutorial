## 세부 의도

사용자는 함수 여러 개가 같은 데이터를 둘러싸고 흩어져 있을 때, 그것들을 하나의 클래스로 모아 응집도를 높이는 리팩토링을 초보자가 체감하길 원한다. 설명은 코드가 아니라 대중 앱에서 보이는 알림, 결제, 추천, 이동 경로 같은 친숙한 기능 묶음의 시각 은유로 전달한다. 모든 핵심 개념은 사용자가 직접 드래그, 전환, 선택, 실행해 보며 이해하도록 구성한다.

## 결정/사유

- AGENTS.md에는 "Introduce Parameter Object"가 남아 있으나, 현재 세션의 명시 요청이 "Combine Functions into Class"이므로 본 TOPIC이 단일 진실원이다.
- 이미지 생성형 정적 비주얼보다 CSS/SVG/Canvas 기반 추상 인터랙션을 채택한다. 사유: 사용자가 직접 조작하며 함수가 클래스로 모이는 변화를 체감해야 하기 때문이다.
