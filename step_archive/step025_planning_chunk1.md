# Step 025 기획

## 제품 한 줄

초보자가 "흩어진 기능 조각이 같은 앱 데이터를 바라볼 때, 하나의 클래스로 모으면 변경 지점이 줄어든다"를 코드 없이 체감하는 인터랙티브 웹 튜토리얼.

## 핵심 은유

- 전: 배달 앱의 거리 요금, 쿠폰, 세금, 예상 시간이 화면 곳곳에서 같은 주문 정보를 따로 확인한다.
- 후: 같은 주문 정보를 바라보는 조각들이 "주문 센터"로 모여 한곳에서 판단한다.
- 사용자는 조각을 직접 모으며 수정 경로, 중복 판단, 확신 지표가 바뀌는 것을 본다.

## 대상 사용자

- 함수, 클래스, 리팩토링을 막 배우는 초보자.
- 코드 문법이 아니라 "왜 묶는가"와 "언제 묶는가"를 먼저 이해해야 한다.

## 대중 앱 사례

- 배달 앱: 주문 요약, 배달비, 쿠폰, 예상 시간.
- 음악 앱: 플레이리스트 길이, 추천, 공유 문구, 분위기 태그.
- 사진 앱: 필터, 밝기, 내보내기 크기, 공유 상태.

## 인터랙션 목록

- 앱 사례 탭: 같은 리팩토링 원리를 다른 대중 앱 맥락에서 본다.
- 기능 조각 선택/드롭: 흩어진 조각을 센터로 모은다.
- 단계 레일: 흩어짐, 공통 데이터 발견, 센터화, 안전 확인을 오간다.
- 안전 체크: 같은 데이터를 보는지, 결과를 유지하는지, 센터 이름이 자연스러운지 확인한다.
- 실시간 지표: 수정 경로와 중복 판단은 낮아지고 확신은 올라간다.

## 코드 비노출 정책

- 화면에 `<code>`, `<pre>`, 코드 조각, 함수 본문, 클래스 본문을 표시하지 않는다.
- "함수"라는 단어는 개념명으로만 쓰고, 문법 예시는 쓰지 않는다.
- 추상 도형, 흐름선, 카드, 지표, 상태 변화로 설명한다.

## 조사 반영

- Fowler 공식 카탈로그: 흩어진 기능이 클래스 안으로 모이는 전후 구조를 은유로 변환.
- Fowler 책 페이지: 작은 행동 보존 변화 원칙을 안전 체크로 변환.
- Material Design: 탭, 클릭, 드롭, 실시간 반응을 기본 조작으로 채택.
- Awwwards: 여백 있는 시각 캔버스와 필터형 탐색을 참고하되 학습 도구 밀도를 우선.

## 검증 기준

- 첫 화면에서 바로 학습 도구가 보여야 한다.
- 모든 핵심 개념은 조작 가능한 상태 변화를 동반해야 한다.
- 빌드, 유닛 테스트, E2E, 스크린샷 확인을 통과해야 한다.
- 학습 화면에는 코드 스니펫이 없어야 한다.
